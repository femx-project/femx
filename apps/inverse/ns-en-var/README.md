# ns-en-var

Navier-Stokes ensemble-variational inverse app.

The optimizer variable is the ensemble coefficient vector `alpha`; the full
physical inverse parameter vector `prm` is reconstructed through the ensemble
basis.

The app evaluates

```text
m = mean + A * alpha
d(alpha) = obs_mean + obs_perturbations * alpha
J(alpha) = 0.5 * alpha_obs * ||d(alpha) - obs||^2
           + 0.5 * prior_weight * ||alpha||^2
```

`obs_mean` and `obs_perturbations` are generated by forward solves before the
optimization. The optimizer itself solves only the small coefficient problem;
it does not solve Navier-Stokes forward or adjoint equations inside each
optimization step. A final forward solve is run after optimization for
validation and visualization.

## Straight-Tube Workflow

After building the three apps, the straight-tube test can be reproduced with:

```bash
FEMX_BUILD_DIR=/path/to/build apps/inverse/ns-en-var/run-straight-tube.sh
```

The script runs `make-ensemble`, `make-obs-ensemble`, and `ns-en-var` in order.
By default it uses 4 MPI ranks for the forward apps and writes logs under
`inputs/straight-tube/output/logs`.

For a heavier check, increase the ensemble and optimizer limits:

```bash
MODES=4 INIT_MODES=4 MAX_ITERATIONS=80 ABS_TOL=1e-10 \
  LOG_DIR=apps/inverse/ns-en-var/inputs/straight-tube/output/logs-heavy-4x4 \
  FEMX_BUILD_DIR=/path/to/build apps/inverse/ns-en-var/run-straight-tube.sh
```

## Ensemble Input

Add an `ensemble` block to the JSON config:

```json
{
  "ensemble": {
    "mean_file": "mean.txt",
    "perturbations_file": "perturbations.txt",
    "obs_mean_file": "obs_mean.txt",
    "obs_perturbations_file": "obs_perturbations.txt",
    "prior_weight": 1.0
  }
}
```

The app uses a precomputed observation-space ensemble and optimizes the
linearized observation model.

`mean_file` is optional. When omitted, the app uses its normal initial inverse
parameter vector as the ensemble mean.

`mean_file` is a whitespace-separated vector with `nprm` values.

`perturbations_file` starts with row and column counts, followed by row-major
entries:

```text
nprm nens
a_00 a_01 ... a_0,nens-1
a_10 a_11 ... a_1,nens-1
...
```

The perturbation row count must match the physical inverse parameter count.

## Ensemble Generation

For the straight-tube sine test, generate a profile-aware perturbation basis
with:

```bash
make-ensemble --config Config.json --modes 2 --initial-modes 2 \
  --amplitude 0.35
```

`--modes` is the number of boundary pulse modes. `--initial-modes` is the
number of independent initial velocity modes. `--amplitude` is the relative
perturbation amplitude. `make-ensemble` writes to `perturbations_file` by
default; `--output` can still override it.

Boundary modes use the configured Poiseuille inlet profile and periodic sine
time functions, so the first boundary mode matches the reference inlet pulse
shape for this test case. Initial velocity modes use the same profile with
low-order axial sine shapes and only fill rows enabled by `controls.init_vel`.

Then generate the observation-space ensemble:

```bash
make-obs-ensemble --config Config.json
```

`obs_mean_file` and `obs_perturbations_file` must point to those generated
files. `make-obs-ensemble` writes to those config paths by default;
`--obs-mean-output` and `--obs-perturbations-output` can still override them.
